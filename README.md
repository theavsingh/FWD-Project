# StudyFlow: A Minimalist Web-Based Academic Planner

StudyFlow is a centralized, front-end web application designed to help students manage academic responsibilities through a distraction-free interface. The platform addresses the challenge of information overload by consolidating fragmented planning tools into a single responsive environment.

## 📋 Project Overview

Modern academic life requires juggling classes, exams, and assignments. StudyFlow provides a lightweight alternative to complex productivity suites by focusing on essential student-centric features with a clean, intuitive UI.

## ✨ Core Features

- **Interactive Study Planner**: A dedicated system for adding academic tasks and tracking completion progress.
- **Live Exam Countdown**: Real-time timers to help students visualize remaining time for upcoming assessments.
- **Academic Image Organization**: A prototype feature designed for the management and sharing of notes, diagrams, and whiteboard captures.
- **Personalized Frontend Experience**: Secure, login-based logic implemented entirely on the client side to maintain user-specific data.
- **Responsive Design**: Optimized for seamless utility across both desktop and mobile devices.

## 🛠️ Technical Implementation

- **Languages**: HTML5 (Semantic Structure), CSS3 (Flexbox/Grid), and JavaScript (ES6+).
- **DOM Manipulation**: Manual logic for task management and live timer updates.
- **Data Persistence**: Utilization of Browser `localStorage` for retaining user information without backend overhead.
- **Deployment**: Hosted via GitHub Pages for easy accessibility.
- **Repository**: [View Source Code](https://github.com/theavsingh/FWD-Project)

## 📈 Methodology

The project follows a phased development lifecycle:
1. **Structural Phase**: Developing the semantic HTML framework for navigation and planner modules.
2. **Styling Phase**: Implementing modern, minimalist layouts using CSS Grid and media queries.
3. **Logic Phase**: Creating JavaScript functions for dynamic task handling and form validation.
4. **Integration Phase**: Ensuring cross-device responsiveness and local data storage reliability.
